# 선수지식 부트캠프 체크리스트

## 학생 체크리스트

- [ ] GET과 POST 차이를 설명할 수 있습니다.
- [ ] 상태 코드 `200`, `201`, `400`, `404`의 뜻을 말할 수 있습니다.
- [ ] JSON body에서 값을 직접 수정할 수 있습니다.
- [ ] `git clone`, `git checkout -b`, `git add`, `git commit`을 따라 할 수 있습니다.
- [ ] branch를 만드는 이유를 한 문장으로 설명할 수 있습니다.
- [ ] table, row, column, PK를 회원 표 예시로 설명할 수 있습니다.

## 강사 / PPT 체크리스트

- [ ] HTTP 메서드 예시가 준비되어 있는가
- [ ] 상태 코드 예시가 준비되어 있는가
- [ ] JSON 요청/응답 예시가 준비되어 있는가
- [ ] Git branch 흐름을 그림이나 단계로 보여줄 수 있는가
- [ ] table / row / column / PK 예시가 준비되어 있는가
- [ ] 다음 시퀀스와 연결되는 이유를 한 장면으로 설명할 수 있는가

## 수업 전 준비 체크

- [ ] Postman 컬렉션 파일을 미리 열 수 있는지 확인했습니다.
- [ ] Git 명령어를 실행할 실습 환경이 준비되어 있습니다.
- [ ] DB 표 예시 자료를 화면에 띄울 준비가 되어 있습니다.
- [ ] 학생에게 보여줄 GET / POST 예시 주소가 정리되어 있습니다.

## 수업 중 질문 체크

- [ ] GET 요청은 어떤 상황에서 쓰는지 물어봤습니다.
- [ ] POST 요청 뒤 상태 코드가 왜 달라질 수 있는지 물어봤습니다.
- [ ] branch를 왜 만드는지 학생이 직접 설명하게 했습니다.
- [ ] PK가 왜 필요한지 row 예시와 함께 설명하게 했습니다.

## 수업 후 복습 체크

- [ ] 학생이 응답 본문과 상태 코드를 함께 읽는지 확인했습니다.
- [ ] 학생이 JSON 값을 바꿨을 때 요청 의미가 달라지는 점을 이해했는지 확인했습니다.
- [ ] 학생이 Git 흐름을 명령어 나열이 아니라 작업 순서로 설명할 수 있는지 확인했습니다.
- [ ] 학생이 DB 기초 용어를 표 예시와 함께 다시 말할 수 있는지 확인했습니다.
